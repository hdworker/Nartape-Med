# 📜 Проект: SVO Narrative Tape (Хронолента информационных сюжетов СВО)
*Документ инициации проекта / Technical Brief v1.0*

---

## 1. 🎯 Общее описание
Система визуализации и анализа информационного поля СВО на основе данных Telegram-каналов. Представляет собой интерактивную временную шкалу («ленту»), где:
- **Ось времени** → хронология событий (день → месяц → год)
- **Толщина ленты** → взвешенная активность обсуждений
- **Ветвление** → расхождение интерпретаций / появление новых сюжетов
- **Тупики** → мёртвые темы, вбросы без развития
- **Цвет/текстура** → тональность, тип сюжета, статус верификации

Цель: дать исследователю, аналитику или ЛПР возможность **реконструировать день-за-днём**, отделять факты от нарративов, отслеживать эволюцию тем и выявлять скрытые тренды.

---

## 2. ⚙️ Функциональные требования

### 2.1. Хронолента (Timeline Core)
- Горизонтальная/вертикальная интерактивная шкала времени
- Толщина сегмента = `Σ(реакции × доверие_канала × уникальность_поста)` за временной срез
- Цветовая кодировка: тональность (-1 → +1), тип сюжета (фронт, ВПК, цифра, гео, инфовойна)
- Автоматическая агрегация по дням/неделям/месяцам/годам с плавным масштабированием

### 2.2. Глубина клика (Drill-Down Hierarchy)
- `Год → Месяц → Неделя → День → Сюжетный кластер → Пост → Факт/Автор/Источник`
- При клике раскрывается панель без перезагрузки:
  - Сводка кластера (ключевые фразы, акторы, гео)
  - Список постов с метаданными (реакции, ссылки, верификация)
  - Дерево перекрёстных ссылок/репостов
  - Карточка факта с статусом `✅/⚠️/❌` и первоисточником

### 2.3. Разрезы и фильтры (Multi-Dimension Slices)
- По фактам: только верифицированные / только спекуляции / только технические
- По авторам/каналам: выбор до 5 источников, сравнение нарративов
- По тематическим аркам: «стена дронов», «бумажный монстр», «цифровая изоляция» и др.
- По акторам/технике/гео: `БАПЛА`, `Матрёшка`, `БахмутТелеком`, `Крым`, `МО РФ` и т.д.
- Комбинирование фильтров с сохранением пресетов

### 2.4. Режим «День-за-днём» (Day View)
- Полная сводка одного дня: доминирующий сюжет, второстепенные ветки, всплески активности
- График тональности в течение дня
- Таблица ключевых постов с возможностью быстрого перехода к исходнику
- Карта взаимных ссылок между каналами за выбранный день

### 2.5. Обработка «тупиковых веток»
- Автоматическая маркировка `is_dead_end = true`, если:
  - Жизненный цикл < 72 часов
  - Нет ≥2 перекрёстных ссылок из независимых каналов
  - Тональность резко экстремальная без фактологического ядра
- Визуально: серый пунктир, скрываются по умолчанию, доступны через фильтр

## 3. 🧩 Компонентная модель архитектуры

| Компонент | Назначение | Связь с требованиями |
|-----------|------------|----------------------|
| **Ingestion & Normalizer** | Сбор постов, дедупликация репостов, UTC-нормализация, обогащение метаданными | Обеспечивает основу для хроноленты и day-by-day |
| **Event Extractor (Rule + ML)** | Выделение акторов, техники, гео, типа события, тональности, фактологического ядра | Питает разрезы, фильтрацию, карточки фактов |
| **Vector Aggregator** | Многовекторное представление: текст + мета + event-frame. Векторная сумма с временным окном | Кластеризация, поиск семантически близких постов |
| **Narrative Clusterer** | HDBSCAN с временными и акторными ограничениями, построение DAG-графов ссылок | Формирует ветки, тупики, сюжетные арки |
| **Storage & Index Engine** | PostgreSQL (JSONB + временные индексы), VectorDB (Qdrant), Redis (кеш), ObjectStorage (медиа) | Поддержка drill-down, быстрых срезов, day view |
| **API Gateway & Query Orchestrator** | FastAPI, маршрутизация запросов, агрегация весов, пагинация, кеш | Связывает фронтенд с бэкендом, обеспечивает производительность |
| **Frontend Visualization** | D3.js / Observable Plot, React/Svelte, панель фильтров, drill-down модалки, экспорт | Реализует ленту, кликабельность, разрезы, day view |

## 4. 💻 Варианты реализации (Tech Stack)

| Слой | MVP (быстрый старт) | Production (масштабируемый) |
|------|---------------------|-----------------------------|
| **Scraper** | `telethon` + `asyncio`, SQLite/PostgreSQL | `telethon` + Kafka stream, Redis dedup |
| **NLP/ML** | `multilingual-e5-large` + `spaCy` rules + `scikit-learn` | Fine-tuned `Saiga-7b` / `RuLLaMA` + `vLLM` inference |
| **Clustering** | `hdbscan` + `numpy` temporal weighting | `cuML` (GPU) + custom temporal-constraint layer |
| **Vector DB** | `ChromaDB` (локальный) | `Qdrant` или `Weaviate` (кластер) |
| **Backend** | FastAPI + SQLAlchemy + Celery | FastAPI + `orjson` + RabbitMQ + Kubernetes |
| **Frontend** | Streamlit / Gradio (прототип) | React + TypeScript + D3.js + Tailwind + Vite |
| **Deploy** | Docker Compose на VPS | Helm + ArgoCD + Prometheus + Grafana |

## 5. 📐 Нефункциональные требования

| Параметр | Требование |
|----------|------------|
| **Производительность** | Рендер года ≤ 2 сек, drill-down до поста ≤ 500 мс, загрузка day view ≤ 1.5 сек |
| **Масштабируемость** | Поддержка ≥ 500k постов, ≥ 100 каналов, параллельные сессии ≤ 50 |
| **Целостность данных** | Immutable лог постов, версионирование кластеров, аудит изменений |
| **Верифицируемость** | Каждый факт/кластер кликабелен до исходного поста + метаданные + статус |
| **Безопасность** | Анонимизация персональных данных, rate-limit API, защита от scraping abuse |
| **UX** | Адаптивность (desktop/tablet), offline-кеш последних 30 дней, экспорт в CSV/PDF |

## 6. 🗺️ Дорожная карта (Phased Delivery)

| Фаза | Срок | Состав | Результат |
|------|------|--------|-----------|
| **MVP-1** | 4–6 нед | Scraper → DB → Базовая лента (толщина/цвет) → Простой drill-down | Работающая хронолента по 3–5 каналам |
| **MVP-2** | 6–8 нед | NLP extraction → HDBSCAN → Dead-end tagging → Фильтры по темам/авторам | Семантическая кластеризация, разрезы |
| **Release v1.0** | 8–10 нед | Day-by-Day view → DAG cross-links → Верификация фактов → Export | Полноценный аналитический инструмент |
| **v2.0+** | по запросу | Human-in-loop верификация → Прогноз трендов → API для внешних систем → Мультиязычность | Enterprise-ready, интеграции, AI-ассистент |
